## WAF payload

| Аспект | Требование |
|--------|------------|
| **Формат** | JSON `[{ "payload", "label" }]` или CSV `payload,label` |
| **Метки (демо)** | `benign`, `sql_injection`, `xss`, `path_traversal`, `command_injection`, `generic_injection` |
| **Файлы** | `sample_payloads.json`, `sample_payloads.csv` |

Образцы **минимальны** — только проверка train/inference.

---

## Сетевые потоки

| Аспект | Требование |
|--------|------------|
| **Формат** | CSV, числовые признаки + `Label` для обучения |
| **Метки (демо)** | `BENIGN`, `HTTP_ATTACK`, `UDP_ATTACK` |
| **Файл** | `sample_network_flows.csv` |

Полный список признаков — `configs/network_anomaly.yaml`.

---

## Подготовка реальных данных

| Шаг | Действие |
|-----|----------|
| 1 | Положите авторизованные выгрузки в `backend/data/raw/` |
| 2 | Нормализуйте колонки; удалите PII, не нужные для обучения |
| 3 | Сформируйте сплиты в `backend/data/processed/` с описанием происхождения |
| 4 | Обучите через `configs/*.yaml`; зафиксируйте метрики в `reports/` и joblib |

---

## Ограничения демо

| Ограничение | Рекомендация |
|-------------|--------------|
| Не отражает production-трафик | Указывайте размер и источник в README / отчётах |
| Нет разнообразия атак | Дополняйте своими корпусами |
| Юридические права | Только данные с согласием владельца |
